Introduction

The GeoTools open source java mapping toolkit is a flexible library of routines and objects that can be used to build interactive web-based applets, as well as stand alone applications.

The GeoTools package is not an application in its own right, however, only a toolkit. So, in order to place a map on the web, or into an application, small amounts of additional code are required.

This package aims to provide a suite of small applets that can either be used 'as is', or as a starting point for more advanced programs. (Alternatively, you might just want to cannibalise bits out of each one).

Overview of files

This directory contains everything you need to get started with GeoTools, including the latest release of GeoTools together with support files, sample maps and example web pages.

In the directory that you find this file in, you will see a series of examples. For each there are three files: an .html, a .java and a .class file (e.g. Example1.html, Example1.java and Example1.class).

You will also find two directories: one called maps, which contains the maps used in the examples, and one called jars, which contains the GeoTools toolkit itself and some support files that it uses.

So, in total, we have:

  • readme.html - this file
  • example1.html - a web page which contains a demo
  • example1.java - the source code for a demo
  • example1.class - the compiled (ready to use) version of a demo
  • ...
    • jars/geotools.jar - The GeoTools toolkit
    • jars/support.jar - Some extra code that supports GeoTools
    • jars/collections.jar - A standard extension from sun for java 1.1
    • maps/statepop.zip - a simple map of the US used by the demos

Requirements

Java 1.1 or later: either on its own or built into a browser. You will find Java built into most modern browsers by default.

Demonstrations

The following demonstrations have been put together with the aim of providing clear examples of GeoTools usage. As a result, there is a minimum of non-GeoTools related code. For example, code for error handling is not as much in evidence as would normally be the case.

If you are a non-programmer, then you will probably only be interested in the html files. Try each demo and see which one does what you need. As the demos progress, they add more features. In order to use your own maps, you will need to modify the tags in the html file. You should not need to worry about the code found in the .java files yet.

Each demonstration listed below takes the following format:

Title - the name of the demo

Link - an active link which will take you to a page which shows the demo running

Description: - a quick overview of what the demo does

Customizing: -a list of things which can be changed from the html tags without the need to re-program anything

Key classes: -a list of GeoTools components that were introduced for the first time in this demo. Don't worry about this if you are not a programmer

Right, on with the show...

Example 1

View

Description:

A very simple applet that shows how a shapefile can be read and displayed in an applet along with a standard toolbar.

Customizing:

The applet tag contains a single param tag ('shapefile') that specifies the relative address of the shapefile to load.

Key classes introduced

geotools.ShapefileReader
geotools.Theme
geotools.Viewer
widgets.ToolBar

Example 2

View

Description:

An extension of Example1 that shows how to set up tooltips

Customizing:

The applet tag contains two tags :
'shapefile' - the relative address of the shapefile to load
'tooltip' - the name of the column to pull the tooltips from

Key classes introduced:

geotools.GeoData

Example 3

View

Description:

An extension of Example2 that shows how to shade a map

Customizing:

The applet tag contains three tags:
'shapefile' - the relative address of the shapefile to load
'tooltip' - the name of the column to pull the tooltips from
'shadeby' - the name of a data column

Key classes introduced:

geotools.Shader
geotools.ClassificationShader
geotools.Key

Example 4

View

Description:

A simple extension of Example3, introducing a new navigation widget

Customizing:

The applet tag contains three tags:
'shapefile' - the relative address of the shapefile to load
'tooltip' - the name of the column to pull the tooltips from
'shadeby' - the name of a data column

Key classes introduced

widgets.ZoomLevelPicker

ImageExample

View

Description:

A simple applet which shows how to use an image layer

Customizing:

The applet tag contains only two tags:
param name="image" value="nameOfImageFile"
param name="bounds" value="x,y,width,height"

Key classes introduced:

uk.ac.leeds.ccg.raster.ImageLayer

GraphApplet Example

View

Description:

A fairly complete applet showing how a map and graph can be linked through a highlight manager.

Customizing:

The applet tag contains a large number of tags:
param name="shapefile" value="nameOfShapefileWithoutExtension"
param name="tooltip" value="nameOfColumn"
param name="groups" value="numberOfGroups"
param name="groupXcol" value="nameOfColumn" // for each group
param name="groupXcolor" value="#rrggbb hex colour reference"
param name="groupXname" value="Description of group"

Key classes introduced:

widgets.PieChart
geotools.GeoLabel
geotools.HighlightManager
